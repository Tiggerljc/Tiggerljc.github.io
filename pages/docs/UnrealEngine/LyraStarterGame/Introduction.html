The Lyra Starter Game is Epic's recommended standard for multiplayer development; as such, understanding the makeup of the project--and, more importantly, the framework it rests on--is crucial to working effectively with Lyra. This page will give an overview of the Lyra framework and the concepts necessary to work with it.

The Basics

Modular Gameplay

Lyra is built as a modular framework composed of Game Feature Plugins (GFPs). These plugins house most of Lyra's gameplay content and systems. Very little is stored in the main Content folder; everything else is stored in GFPs and loaded dynamically at runtime.

The default Lyra plugins include:

  • ShooterCore
  • ShooterExplorer
  • ShooterMaps
  • ShooterTests
  • TopDownArena

This modular architecture allows for a clean separation of gameplay systems and enables flexible game modes and content loading, meaning that only the necessary content is available at runtime.

Gameplay Ability System

Lyra makes use of the Gameplay Ability System (GAS) for player actions. Any action that the player character carries out--for example, running, jumping, and dashing--is controlled by a Gameplay Ability. These abilities are modular and can be granted or removed from the player at runtime through Ability Sets.

Input is handled using Input Actions, which are mapped to abilities using Input Mapping Contexts (IMCs).

Another key part of Lyra's ability system is the use of Gameplay Tags. These tags govern ability relationships, activation and deactivation, when abilities can or cannot happen, what abilities block other abilities, player health and damage, and more.

Experiences

Lyra uses Gameplay Experiences to create game modes. These Experiences define which Game Features to enable, what pawn to use, and what abilities to grant that pawn. Lyra takes this a step further by creating the LyraUserFacingExperienceDefinition, which takes in an Experience Definition and assigns it a map, title/subtitle, description, and logo, as well as a loading screen widget.

The LyraUserFacingExperienceDefinition is used in the L_DefaultEditorOverview map when constructing B_ExperienceList3D. This asset loads the Experience Definitions as widgets, and creates a platform for the player to walk onto. This platform loads a Gameplay Experience, including its associated map and rulesets. This example could be taken to assemble a full lobby screen with many different game modes.
