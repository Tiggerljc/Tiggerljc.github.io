Built on Lyra!

Lyra is so incredibly dense that it is hard to work with, and what's worse: there is very little documentation. Nonetheless, my game project is built off of the Lyra framework. Therefore, I am studying Lyra's architecture, and this page is where I document an in-depth breakdown of the Lyra Starter Project.

Core Principles

When working with Lyra, it is best to adhere to some core principles. Doing so can save you a lot of frustration in the long run. Wherever possible, always extend code instead of modifying it. First and foremost, if Epic ever updates Lyra, it makes it much easier to merge your code with the updated code. This also makes your code more modular, which is the entire goal of Lyra.

Additionally, content should be housed in Game Feature Plugins (GFPs). This aids in modularizing your code and keeps it from interfering with Lyra's core content. Also, it is the system Lyra itself uses to create its own content, and you should always prefer using Lyra's systems over creating your own.

Getting Started with Lyra

Prerequisites: Please refer to the Epic documentation to become familiar with the basics of Lyra. This breakdown will be more of a more low-level in-depth examination, so you will likely need to know some of the basics before reading this.

Basic File Structure

Lyra's content is divided into three main groups; the Content folder, core C++, and plugins.

The Content folder houses defaults for the game framework (general global Blueprints and Data Assets, etc.); this is not where game content should go.

  • Content (Engine Content folder)

The core C++ content includes LyraEditor and LyraGame. LyraEditor is mostly for Editor- and PIE-related setup, and LyraGame is the C++ side of the game framework. It is very important to understand LyraGame as most content is dependent upon it.

  • C++ Classes
    • LyraEditor
    • LyraGame

Game Feature Plugins house all of Lyra's actual game content; the experiences, the maps, etc. are in GFPs. The main one is ShooterCore, which holds the assets and definition for most of the actual game modes.

  • Game Feature plugins
    • ShooterCore (Core game plugin)
    • Shooter Explorer
    • Shooter Maps
    • Shooter Tests
    • Top Down Arena

Core Frameworks

The Lyra Starter Game is built off of multiple modular asset plugins, such as Game Feature Plugins and the Gameplay Ability System. These serve to componentize the project and ensure that everything is only loaded when it needs to be.
