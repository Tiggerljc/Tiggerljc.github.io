Content

Lyra is a very modular project, which means that most of the game content is loaded in plugins at runtime rather than put into a content folder. However, it's core (as in, necessary for the game itself to run) assets are stored in the main Content folder. This content is used to create the frontend and stores the base character asset and it's respective ability sets, as well as some UI components, audio assets, weapon meshes, and Blueprints.

This page will give a high-level overview of all of the folders and files in the Content folder, and the more important assets will get their own seperate pages to explain them more in-depth.

Files

B_LyraGameInstance

B_LyraGameInstance is a subclass of ULyraGameInstance. It inherits the functions from the parent class, and has two audio components, MusicSystem and Audio_LobbySelect (probably for the menu/lobby music). It's main purpose is to expose the C++ GameInstance into the Editor so it can be assigned in Project Settings.

B_LyraGameMode

B_LyraGameMode is a Blueprint extension of the C++ class ALyraGameMode. This asset runs on the server side. It's main responsibility is to expose the class defaults so they can be set in Blueprint. These defaults include:

  • LyraGameSession
  • LyraGameState
  • LyraPlayerController
  • LyraPlayerState
  • LyraHUD
  • LyraCharacter
  • SpectatorPawn
  • LyraReplayPlayerController
  • ServerStatReplicator

DefaultGame_Label

DefaultGame_Label is a Primary Asset Label with the parent UPrimaryAssetLabel. It allows you to group assets for the asset manager and defines how those assets are cooked/loaded.

DefaultGameData

DefaultGameData is a Data Asset with the parent class LyraGameData. It has three parameters that take in Gameplay Effect assets. These abilities are used by default in Lyra for damage, healing, and dynamic tag application. The assets are as follows:

  • Damage Gameplay Effect (SetByCaller): GE_Damage_Basic_SetByCaller
  • Heal Gameplay Effect (SetByCaller): GE_Heal_SetByCaller
  • Dynamic Tag Gameplay Effect: GE_Dynamic Tag

The C++ UCLASS ULyraGameData associated with this asset has three properties, which line up with the three parameters in the Data Asset.

Folders

Audio

The Audio folder appears to hold all of the audio for the entire game. This does not mean that you should store your audio files here; they should go in your own content plugins. It has a bunch of folders for different sound effects, audio mixes, etc. There's not too much to see here, and I don't know much about audio in Unreal Engine anyways {guilty shrug}.

Characters

The Characters folder holds the default character data for Lyra, the third-person camera, and the meshes for the UE4 and UE5 mannequins. This folder also contains the "simple pawn" used in the L_DefaultEditorOverview map (the map open by default in a fresh Lyra project), as well as the ability sets for the default character class.

Also in this folder is the Character_Default Blueprint, whose parent class is ALyraCharacter. This Blueprint simply inherits the Lyra Character class and sets up the capsule collision component for the character.

The Heroes subfolder contains the B_Hero_Default Blueprint. It is a child of the Character_Default Blueprint, and it's the base hero class for Lyra. It contains a few Blueprint functions that handle team coloring, ragdoll animations, and death state visuals.

ContextEffects

This folder holds only three assets; CFX_DefaultSkin, DT_AnimEffectTags, and DT_SurfaceTypes. These three assets are for defining tag relationships relative to different surfaces, and telling the Engine what effects to apply to the character based on these tags. In this case, they are used to define the different sounds produced by walking, landing, jumping, or jogging on a specific surface—however, the system can be used for any sound or visual effect (for example, Niagra spark particles drifting off of the player when walking on a hot surface).

The only surfaces defined by default are Default, Character, Concrete, and Glass.

Editor

This folder is completely empty in-editor; a quick glance in Explorer shows that contains a single file (CheckContent.svg). This file lives in Slate/Icons, meaning that is a Slate vector icon used somewhere in the Editor... it appears to be for the content validation button located at the top of the main Editor window.

Effects

The Effects folder is home to most of the particle systems as well as some animation notifies and material functions. Everything from launchpad particles to damage effects on the camera when hit to the floating damage numbers when an attack lands.

Note: the AnimationNotifies folder is marked for deprecation.
